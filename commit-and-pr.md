# Commit & PR Protocol

## Commits
- Prefix: `feat: `, `fix: `, `docs: `, `chore: `, `refactor: `, `dev: `.
- Subject: Imperative Title Case.
- Body Format:
  ```text
  Before, <previous state or problem description>.
  Now, <new state or outcome from the user's perspective>.
  ```

## Pull Requests
- Format: Must be formatted like commits above
- Describe what roles (Traveler/Follower) gain or lose explicitly across all commits in this PR.
- Unified Narrative: Write a story about the change, not just technical bullets.
- Pull Request must describe the objectives of the PR as a whole, not just the most recent changes in the branch.
